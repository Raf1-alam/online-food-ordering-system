🔴 Critical — Missing for Real Usability

7 features
#1
Coupon / Promo Code Field at Checkout
What exists:GET /api/v1/coupons/validate/{code} — validates a code and returns a discountPercentage.
What's missing:The Checkout page has NO coupon input field. Users cannot apply discounts even though the infrastructure is fully built.
Fix:Add a coupon code input to Checkout.tsx that calls the validate endpoint and subtracts the discount from the cart total before submission.
#2
Favorites Page / UI — Button Not Wired to Backend
What exists:POST /api/v1/favorites/{restaurantId} (toggle) and GET /api/v1/favorites (returns set of IDs).
What's missing:The heart button in RestaurantList.tsx is purely client-side state — favorites are never saved to the database. No "My Favorites" page exists.
Fix:Wire the heart button to the backend toggle endpoint. Add a "My Favorites" page that loads the user's saved restaurants.
#3
Real Restaurant Images
What exists:Every restaurant card shows the exact same hardcoded Unsplash photo URL.
What's missing:Restaurant-specific images — the Restaurant entity has no imageUrl field. No upload mechanism exists.
Fix:Add imageUrl to Restaurant entity, expose it in request/response DTOs, let staff set it during application or in their profile. Display it on cards.
#4
Menu Item Images Not Displayed
What exists:MenuItem entity has an imageUrl field and it is stored in the DB.
What's missing:Nowhere on the menu page are item images rendered — the field is simply ignored by the UI.
Fix:Render menuItem.imageUrl in the menu item card. Fall back to a placeholder icon if null.
#5
Order Cancellation by Customer
What exists:A CANCELLED status exists in the state machine and is displayed on the Staff Dashboard history tab.
What's missing:Customers cannot cancel their own order. There is no "Cancel Order" button in MyOrders.tsx for eligible orders in PLACED state.
Fix:Add a cancel button visible only when status === 'PLACED'. Add a backend endpoint that transitions the order to CANCELLED.
#6
Admin: Coupon Management UI
What exists:The Coupon entity (code, discountPercentage, expiryDate, active) and a validation-only endpoint.
What's missing:Admins have no way to create, list, or deactivate coupons through the UI. No admin CRUD endpoints exist for coupons.
Fix:Add Coupon Management tab to Admin Dashboard with list, create form (code / discount% / expiry date), and deactivate toggle. Add backend CRUD endpoints.
#7
About Us & Contact Pages (404)
What exists:Navbar links labeled "About Us" and "Contact" are visible to guests and customers.
What's missing:Clicking either link leads to a 404 — no route or page content exists.
Fix:Create simple static pages for /about and /contact with team information and a contact form.

🟡 Important — Significant UX Gaps

9 features
#8
Email Confirmations & Status Notifications
What's missing:No email is sent when an order is placed or when its status changes. Users cannot trust the system — there is no paper trail.
Fix:Integrate Spring Mail (JavaMailSender) to send order confirmation and status update emails to customers.
#9
Forgot Password / Password Reset Flow
What's missing:If a user forgets their password there is no recovery mechanism — only an admin can manually reset accounts.
Fix:Add "Forgot Password" on the login page. Backend generates a short-lived reset token, sends a reset email, and exposes a token-authenticated reset endpoint.
#10
Saved Delivery Addresses (Address Book)
What's missing:Every order requires the customer to type their full delivery address from scratch. Real platforms save addresses (Uber Eats, Foodpanda).
Fix:Add an addresses table linked to users. Allow saving 3–5 addresses. Show a dropdown at checkout to select a saved address or enter a new one.
#11
Restaurant Profile Editor (Staff)
What's missing:Staff can manage their menu but cannot edit their restaurant's own public profile — name, description, address, phone, image, cuisine type, or opening hours.
Fix:Add a "Restaurant Settings" tab to the Staff Dashboard with a form that loads and saves the restaurant's public details via PUT /api/v1/restaurants/{id}.
#12
Operating Hours / Open-Closed Status
What's missing:Restaurants have no operating hours. A customer can attempt to order from a restaurant at 3 AM even if it's closed.
Fix:Add openingTime / closingTime fields to Restaurant. Show an "Open / Closed" badge on restaurant cards. Prevent checkout from closed restaurants.
#13
Estimated Delivery Time (ETA)
What's missing:Once an order is placed, customers have no idea when to expect it. There is no ETA shown anywhere in the UI.
Fix:Store estimatedDeliveryTime in the Order entity, set by staff when confirming. Display it prominently in MyOrders.tsx for active orders.
#14
Menu Item Categories & Filtering Tabs
What's missing:The menu page shows all items in a flat list with only a text search. Real menus group items by category (Starters, Mains, Desserts, Drinks).
Fix:Add a category field to MenuItem. Group the menu display by category tabs. Support category assignment in MenuManagement.
#15
Cart Item Quantity Editing
What's missing:Once an item is in the cart, users cannot change the quantity in the sidebar — they must remove and re-add the item entirely.
Fix:Add +/− buttons next to each cart item in CartSidebar.tsx that call PUT /api/v1/cart/items/{itemId} with the updated quantity.
#16
Reorder from Previous Orders
What's missing:Users with previous orders have no way to quickly reorder the same items. Past orders show items but there is no "Reorder" shortcut.
Fix:Add a "Reorder" button to completed order cards in MyOrders.tsx that pre-fills the cart with the same items from the same restaurant.

🟢 Nice-to-Have — Competitive Differentiators

9 features
#17
Real-Time Order Tracking Map
Currently only a status tracker bar. Real apps show a live map with the delivery rider's GPS location. Requires GPS tracking infrastructure — high complexity, very high impact.
#18
Search by Food Item (Cross-Restaurant)
Users can only search for restaurants. They cannot search for a specific dish like "Chicken Biryani" and see all restaurants that serve it. Requires a global search endpoint across all active menu items.
#19
Restaurant Rating Breakdown Widget
Restaurant cards show a single average rating number. No breakdown of how many 5-star, 4-star, etc. reviews exist. Add a mini bar chart of rating distribution on the menu page.
#20
Loyalty Points / Rewards System
No incentive to return. Real apps (Zomato, Foodpanda) give coins per order. Add a loyaltyPoints field to User. Award points on successful delivery. Allow redeeming points as a discount at checkout.
#21
Social Sharing
No way to share a restaurant or menu item. Add share buttons (copy link, WhatsApp, Facebook) to restaurant and menu pages to drive organic traffic.
#22
Admin Dashboard Analytics & KPIs
The admin sees only raw data tables. No charts or KPIs — total revenue this month, top restaurants, most-ordered items, new user signups. Build an analytics panel with Recharts or Chart.js.
#23
Staff: Daily Sales Summary Card
Staff can see live orders and history, but no summary of daily revenue or most popular items. Add summary cards at the top of the Staff Dashboard showing today's order count and total revenue.
#24
Menu Item "Popular" / "Chef's Recommended" Labels
All menu items look identical. Add boolean isPopular / isRecommended fields to MenuItem. Render a highlighted badge on marked items to guide customer decisions.
#25
Dietary & Allergen Tags on Menu Items
No dietary information on menu items (Vegetarian, Vegan, Gluten-Free, Spicy, Halal). Critical for real-world usability. Add a tags array field to MenuItem and render colored tag badges on each item card.
